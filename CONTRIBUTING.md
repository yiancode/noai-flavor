# 贡献指南

感谢你对 noai-flavor 的关注！这个项目靠社区贡献不断完善。

## 可以贡献什么

### 1. 新的反模式

发现了新的 AI 写作特征？欢迎提交！

**提交格式**（在 `patterns/` 目录下创建新文件）：

```markdown
# [反模式名称]

## 特征描述
[简要说明这个反模式的特点]

## 反模式示例

❌ **AI 味写法**：
> [具体例子]

## 自然写法

✅ **改进后**：
> [改进后的例子]

## 规则（供 AI 参考）
- [规则 1]
- [规则 2]
```

### 2. 平台特定指南

熟悉某个平台的写作风格？可以补充：
- 平台特有的表达习惯
- 该平台用户的阅读偏好
- 针对性的 Prompt 调整

### 3. 对比示例

真实的改写案例特别有价值：
- 原始 AI 输出
- 改写后的版本
- 改写思路说明

### 4. 工具集成

使用其他 AI 工具？欢迎补充集成指南。

## 提交流程

1. Fork 这个仓库
2. 创建你的分支：`git checkout -b feat/new-pattern`
3. 提交改动：`git commit -m 'add: 新反模式名称'`
4. 推送分支：`git push origin feat/new-pattern`
5. 创建 Pull Request

## 写作风格

既然是教人写得自然，我们自己的文档也要自然：

- 少用 emoji（讽刺的是）
- 别套话，直接说
- 例子要具体，避免抽象描述
- 中文为主，术语可用英文

## 问题反馈

有问题或建议？直接开 Issue，描述清楚就行。
