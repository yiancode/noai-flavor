# 设计工具推荐

去除 AI 味儿的关键方法之一是**设计优先开发**。

## 核心思路

不要上来就让 AI 梭哈整个项目，而是：

1. **先让 AI 做个前端 Demo（纯静态页面）**
2. **对设计满意后，再让 AI 基于 Demo 开发完整项目**

这样可以避免做完后发现设计不对，需要重新返工。

## 设计工具

### Google Stitch

**特点**：
- Google 推出的 AI 设计工具
- 输入描述，生成专业界面原型
- 支持手绘草图识别
- 可以人工修改主题风格
- 导出代码或设计文件

**使用流程**：

1. 输入一段描述："一个极简风格的博客首页"
2. AI 生成界面原型
3. 人工调整主题（颜色、字体、间距）
4. 或者标注要修改的部分，让 AI 调整
5. 导出代码或下载设计文件
6. 把代码或设计稿交给 Cursor/Claude Code 继续开发

**适合场景**：
- 快速原型设计
- 从零开始的项目
- 不会用 Figma 的开发者

**官网**：https://stitch.google.com

---

### Figma + Figma MCP

**特点**：
- 业界标准的设计工具
- 精细控制设计细节
- 配合 Figma MCP，AI 可以直接读取设计文件

**使用流程**：

1. 在 Figma 中设计网站界面
2. 安装 Figma MCP 扩展
3. 让 AI 读取 Figma 设计文件
4. AI 按照设计生成代码

**提示词示例**：
```
请读取 Figma 设计文件：[Figma URL]
按照设计精确还原界面。
```

**适合场景**：
- 需要精细设计
- 团队协作
- 设计师 + 开发者配合

**官网**：https://www.figma.com

---

### Onlook

**特点**：
- 号称"为设计师准备的 Cursor"
- 可视化编辑网页代码
- 设计和开发无缝衔接
- 所见即所得

**使用流程**：

1. 用 Onlook 打开现有网页
2. 可视化编辑（移动元素、改颜色、调间距）
3. 自动生成代码
4. 保存到代码库

**适合场景**：
- 设计师不想写代码
- 快速调整样式
- 原型迭代

**官网**：https://onlook.dev

---

### Screenshot to Code

**特点**：
- 截图直接转代码
- 支持识别网站布局和样式
- 生成 HTML/CSS/React 代码

**使用流程**：

1. 找到喜欢的网站，截图
2. 上传到 Screenshot to Code
3. 工具识别布局和样式，生成代码
4. 把代码交给 AI 继续开发

**提示词示例**：
```
以下是一个网站的代码，请参考这个设计风格：
[粘贴代码]

不要完全照搬，但要保持相似的视觉风格。
```

**适合场景**：
- 参考现有网站设计
- 快速还原设计
- 学习优秀设计的实现

**官网**：https://screenshottocode.com

---

### V0.dev

**特点**：
- Vercel 出品的 AI 设计工具
- 输入描述，生成 React 组件
- 代码质量高，符合最佳实践
- 可以直接集成到项目

**使用流程**：

1. 输入描述："一个带搜索框的导航栏"
2. V0 生成 React 组件代码
3. 可以在界面上直接编辑
4. 复制代码到项目中

**适合场景**：
- React 项目
- 快速生成组件
- 学习 React 最佳实践

**官网**：https://v0.dev

## 使用流程

### 流程 1：从零设计

```
1. 用户描述需求
   ↓
2. 使用 Google Stitch 或 V0.dev 生成原型
   ↓
3. 人工调整设计（颜色、布局）
   ↓
4. 导出代码
   ↓
5. 交给 Cursor/Claude Code 继续开发功能
```

### 流程 2：参考现有设计

```
1. 找到喜欢的网站
   ↓
2. 使用 Screenshot to Code 转成代码
   ↓
3. 把代码交给 AI，让它参考这个风格
   ↓
4. AI 生成类似风格的网站
```

### 流程 3：套用模板

```
1. 从 HTML5 UP 或 Start Bootstrap 下载模板
   ↓
2. 把模板代码交给 AI
   ↓
3. 让 AI 基于模板修改内容和功能
```

## 提示词模板

### 使用 Figma 设计

```
请读取我的 Figma 设计文件：
[Figma URL]

按照设计稿精确还原界面：
- 严格遵守配色方案
- 严格遵守间距和布局
- 严格遵守字体选择

不要做任何自由发挥。
```

### 参考网站设计

```
请访问以下网站，提取它的设计风格：
[网站 URL]

然后生成类似风格的网站，包括：
- 配色方案
- 布局风格
- 字体选择
- 动画效果

不要完全照搬，但要保持相似的视觉感受。
```

### 使用模板

```
以下是一个网站模板的代码：
[粘贴代码]

请基于这个模板：
1. 保持整体设计风格
2. 修改内容为：[你的内容]
3. 添加功能：[你需要的功能]

不要改变配色和布局风格。
```

## 案例对比

### ❌ 传统做法

```
用户："帮我做一个包含用户系统、后台管理的完整 SaaS 平台"
AI：哗啦哗啦生成几十个文件
结果：页面风格不对，需要返工，浪费 Tokens
```

### ✅ 设计优先

```
用户："先帮我设计 SaaS 平台的首页，纯静态页面"
AI：生成首页 HTML/CSS
用户：查看效果，不满意，调整配色
AI：重新生成
用户：满意了，"好的，现在基于这个设计开发完整项目"
AI：按照已定义的风格开发
```

节省时间，避免返工。

## 工具对比

| 工具 | 适合场景 | 优点 | 缺点 |
|------|----------|------|------|
| Google Stitch | 快速原型 | 简单易用，支持手绘 | 精细度不如 Figma |
| Figma | 精细设计 | 专业、功能强大 | 学习曲线陡峭 |
| Onlook | 可视化编辑 | 所见即所得 | 新工具，功能有限 |
| Screenshot to Code | 参考现有网站 | 快速还原设计 | 识别准确度有限 |
| V0.dev | React 组件 | 代码质量高 | 仅支持 React |

---

## 总结

**核心原则**：
1. 不要让 AI 一次做完所有事情
2. 先设计，后开发
3. 对设计满意了，再继续开发功能
4. 使用工具加速设计流程
5. 参考优秀网站，不要从零开始
