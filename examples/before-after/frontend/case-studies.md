# 前端开发案例对比

本文档展示如何使用 noai-flavor 提示词去除网站的 AI 味儿。

## 案例 1：个人技术博客

### ❌ 优化前（一眼 AI）

**提示词：**
```
开发个人技术博客网站首页
```

**生成的代码特征：**
- 蓝紫色渐变背景（#6366F1 → #8B5CF6）
- Hero + 三卡片布局
- Emoji 作为图标（📝✍️💡）
- 空洞文案："分享我的技术见解"
- 使用 Shadcn UI 默认样式

**问题：**
一眼就能看出是 AI 生成的，毫无个性。

---

### ✅ 优化后（去除 AI 味儿）

**提示词：**
```
开发个人技术博客网站首页

请严格遵守 AGENTS.md 中的设计规则。

额外要求：
- 风格：极客范儿、技术感
- 配色：深灰 (#1A1A1A) + 橙色 (#FF6B35)
- 字体：JetBrains Mono（代码感）
- 图标：使用 Iconify，不用 Emoji
- 插画：使用 unDraw
- 布局：不对称，左对齐文案

参考风格：GitHub、Linear 的深色模式
```

**生成的代码特征：**
- 深灰色背景 + 橙色强调
- 不对称布局，左对齐
- 使用 Iconify 矢量图标
- 具体文案："500+ 篇技术笔记，10 年编程经验"
- 代码感字体（JetBrains Mono）

**效果：**
完全不像 AI 生成的，有个人风格。

## 案例 2：SaaS 产品落地页

### ❌ 优化前（一眼 AI）

**提示词：**
```
开发 SaaS 产品《服务器运维监控平台》的落地页
```

**生成的代码特征：**
- 蓝紫渐变背景
- 完美居中的 Hero Section
- 三个并排卡片介绍功能
- Emoji 图标（🚀📊🔒）
- 空洞文案："提升运维效率"、"实时监控"

**问题：**
典型的 AI 生成模板，没有产品特色。

---

### ✅ 优化后（去除 AI 味儿）

**提示词：**
```
开发 SaaS 产品《服务器运维监控平台》的落地页

请严格遵守 AGENTS.md 中的设计规则。

额外要求：
- 风格：黑客帝国、代码雨
- 配色：黑色 + 绿色（矩阵风格）
- 必须使用 Aceternity UI：https://ui.aceternity.com/components
- 使用 BackgroundBeams 或代码雨效果
- 文案具体化："7×24 小时监控，99.99% 可用性"
- 图标：Iconify 的 mdi 系列

参考氛围：《黑客帝国》的视觉风格
```

**生成的代码特征：**
- 深色背景 + 代码雨效果
- 使用 Aceternity UI 的炫酷组件
- 交错布局（图文左右交替）
- 具体数据文案
- 矢量图标代替 Emoji

**效果：**
专业、有科技感，完全不像 AI 生成。

## 关键差异总结

| 维度 | ❌ AI 味儿重 | ✅ 去除 AI 味儿 |
|------|-------------|----------------|
| **配色** | 蓝紫渐变 | 深灰+橙色 / 深色+冷蓝 |
| **布局** | Hero + 三卡片 | 不对称、交错、大留白 |
| **图标** | Emoji（🚀⚡） | Iconify 矢量图标 |
| **图片** | 灰色占位图 | Pexels 真实照片 |
| **文案** | 空洞（"提升效率"） | 具体（"节省 2 小时"） |
| **组件库** | Shadcn 默认样式 | Aceternity UI / Magic UI |
| **字体** | Inter / Roboto | JetBrains Mono / Space Grotesk |

## 使用建议

### 1. 使用 AGENTS.md 文件

将 [`prompts/frontend/anti-ai-web.md`](../prompts/frontend/anti-ai-web.md) 的内容保存为 `AGENTS.md`，放在项目根目录。

AI 编程工具会自动读取并遵守规则。

### 2. 明确风格要求

不要只说"做个网站"，而要说：
- 风格倾向（科技/优雅/活力）
- 配色方案（具体色值）
- 参考对象（具体网站或氛围）

### 3. 指定组件库和资源

明确告诉 AI：
- 使用什么组件库（Aceternity UI、Magic UI 等）
- 从哪里获取图片（Pexels、unDraw、Iconify）
- 不要什么（蓝紫色、Emoji、占位图）

### 4. 语境注入

给 AI 一段文字感受氛围：
```
先阅读这段话感受氛围：
《黑客帝国》中的代码雨场景

然后设计一个服务器监控平台，带有这种科技感和神秘感。
```

### 5. 迭代优化

如果第一次生成的效果不满意：
1. 不要直接让 AI 重做
2. 明确指出问题："配色还是太像 Tailwind 默认色"
3. 给出具体要求："改用深灰 #1A1A1A + 橙色 #FF6B35"

## 完整提示词模板

```markdown
开发 [项目类型] 的 [页面类型]

请严格遵守 AGENTS.md 中的设计规则。

## 风格要求
- 风格：[科技感/优雅/活力/极简]
- 氛围：[具体描述或参考电影/艺术作品]

## 配色方案
- 主色：#[HEX 色值]
- 辅助色：#[HEX 色值]
- 背景：#[HEX 色值]
- 文字：#[HEX 色值]

禁止使用 #6366F1、#8B5CF6 等蓝紫色。

## 组件库
必须使用 [Aceternity UI / Magic UI / ikun-ui / Radix UI]
文档：[组件库官网链接]

不要使用 Shadcn UI / Material UI 的默认样式。

## 图片资源
- 图标：使用 Iconify (https://iconify.design)
- 插画：使用 unDraw (https://undraw.co)
- 照片：使用 Pexels (https://www.pexels.com)，搜索关键词 "[相关主题]"
- 占位图：使用 Picsum Photos (https://picsum.photos)

不要使用 Emoji 作为图标。

## 文案要求
- 具体化：用数字、场景、对比
- 口语化：像朋友聊天
- 带情绪：可以幽默、挑衅
- 短句：每句不超过 15 个字

不要用"提升生产力"、"卓越体验"等空洞词汇。

## 布局要求
- 不对称布局
- 大胆留白
- 不要 Hero + 三卡片模式

## 参考风格
- [参考网站 1]
- [参考网站 2]
- 氛围参考：[电影/艺术作品]
```
